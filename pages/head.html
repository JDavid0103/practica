Encabezado y metadatos

<head> 
  <title>Ejercicio</title>
  <meta http-equiv="Content-Type" content="text/html;charset=UTF-8">
  <meta name="description" content="Pagina web de ejercicios">
  <meta name="author" content="José David">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta http-equiv="refresh" content="90">
  <meta http-equiv="Content-Security-Policy" content="default-src https:" />
  <meta http-equiv="set-cookie" content="">
</head>

> Etiqueta title

La etiqueta <title> define el título del documento HTML. Este texto aparece en la pestaña del navegador, en los resultados de los buscadores y al guardar la página en marcadores. Solo puede haber un <title> por documento.

> Meta Content-Type

La etiqueta <meta http-equiv="Content-Type" content="text/html;charset=UTF-8"> indica el tipo de contenido y el conjunto de caracteres que usa la página. En este caso, se especifica que el documento es HTML y usa la codificación UTF-8, que permite mostrar correctamente caracteres especiales como tildes o la letra ñ.

> Meta description

La etiqueta <meta name="description" content="..."> proporciona una breve descripción del contenido de la página. Los motores de búsqueda utilizan este texto para mostrar un resumen en los resultados, mejorando el SEO (posicionamiento web).

> Meta author

La etiqueta <meta name="author" content="..."> indica el nombre del autor o creador del documento. Es útil para información de créditos o identificación del responsable del sitio.

> Meta viewport

La etiqueta <meta name="viewport" content="width=device-width, initial-scale=1.0"> se utiliza para adaptar la visualización de la página a diferentes dispositivos (como móviles o tablets). Controla el tamaño y la escala inicial del área visible.

> Meta refresh

La etiqueta <meta http-equiv="refresh" content="90"> recarga automáticamente la página cada 90 segundos. Se puede usar también para redirigir a otra URL tras un tiempo determinado, aunque se recomienda usar JavaScript o cabeceras HTTP modernas para ello.

> Meta Content-Security-Policy

La etiqueta <meta http-equiv="Content-Security-Policy" content="default-src https:" /> establece una política de seguridad del contenido (CSP) para el navegador. En este caso, limita la carga de recursos solo a fuentes seguras (HTTPS), ayudando a prevenir ataques como la inyección de scripts maliciosos.

> Meta set-cookie

La etiqueta <meta http-equiv="set-cookie" content=""> intentaba antiguamente establecer cookies desde el encabezado del documento, pero hoy en día está obsoleta y no es compatible con la mayoría de los navegadores. La gestión de cookies debe hacerse mediante cabeceras HTTP o JavaScript del lado del servidor.
